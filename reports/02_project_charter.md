# Project Charter
## Customer Segmentation using Machine Learning for Jumia

## Project Overview

This project will develop a customer segmentation framework for Jumia using customer transaction behavior as the primary source of insight. The work will follow the CRISP-DM methodology and will focus on understanding customer value, identifying risk, and grouping customers into meaningful segments that can support smarter marketing and retention decisions.

The project is designed as a  capstone initiative and is intended to resemble professional consulting work. It will combine business analysis, machine learning, and clear documentation so that the outcomes are useful to both academic reviewers and business stakeholders.

## Goals

The main goals of the project are to:

- understand customer behavior through transaction data,
- identify high-value and at-risk customer groups,
- create a practical segmentation model using clustering methods,
- and translate analytical findings into business recommendations.

## Objectives

The project will achieve the following objectives:

1. Translate the business need into a formal data science problem.
2. Define customer value and customer risk clearly.
3. Apply RFM analysis to create customer-level behavior features.
4. Use clustering to discover customer segments.
5. Present business-friendly insights and recommendations.

## Scope

The project scope includes:

- reviewing the business problem and project brief,
- preparing and exploring the dataset,
- creating RFM-based features,
- applying clustering techniques,
- evaluating the resulting segments,
- and documenting findings in a professional format.

## Out of Scope

The following items are outside the scope of this project:

- live deployment to a production CRM system,
- integration with Jumia's real operational databases,
- full customer lifetime value modeling,
- and advanced personalization systems.

## Deliverables

The project will deliver the following outputs:

- a business understanding document,
- a project charter,
- a GitHub repository with documentation and analysis,
- a weekly progress tracker,
- and a final presentation of findings and recommendations.

## Budget Assumptions

The project is expected to use open-source tools and school-supported computing resources. No major commercial software purchase is required. The main budget assumptions are:

- no paid cloud infrastructure required,
- no external data purchase required,
- and use of freely available Python libraries and documentation tools.

## Dependencies

The project depends on the following:

- access to the Online Retail Dataset,
- Python and data analysis libraries,
- documentation tools for Markdown and GitHub,
- and a clear project timeline for completion.

## Success Metrics

The project will be considered successful if it delivers:

- a clear business understanding document,
- a practical segmentation model,
- understandable customer segments,
- and recommendations that can be used by a business team.

## Approval Criteria

The project will be approved for completion when the following are satisfied:

- the business problem is clearly defined,
- the analysis is complete and documented,
- the segmentation findings are interpretable,
- and the final deliverables meet academic and professional standards.
